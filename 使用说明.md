## 1. 快速开始 - 推送到 GitHub 🚀

### 第一步：在 GitHub 上创建仓库

1. 登录你的 GitHub 账号：https://github.com
2. 点击右上角 **+** 号 → 选择 **New repository**
3. **仓库名称必须是**：`你的用户名.github.io`
   - 例如：如果你的 GitHub 用户名是 `Luowenbo`，仓库名就是 `Luowenbo.github.io`
4. 选择 **Public**（公开仓库，免费）
5. ❌ **不要**勾选 "Add a README file"（我们已有）
6. 点击 **Create repository**

### 第二步：推送代码

打开 PowerShell 或命令提示符，依次执行以下命令：

```bash
# 进入网站目录
cd f:\aca

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交文件
git commit -m "Initial commit: academic website"

# ⚠️ 连接到你的 GitHub 仓库（把 Luowenbo 换成你的用户名）
git remote add origin https://github.com/Luowenbo/Luowenbo.github.io.git

# 推送到 GitHub
git push -u origin main
```

### 第三步：启用 GitHub Pages

1. 进入你的 GitHub 仓库页面
2. 点击 **Settings**（设置）
3. 在左侧菜单找到 **Pages**
4. **Source** 部分：
   - Branch 选择 `main`
   - Folder 选择 `/ (root)`
5. 点击 **Save**

✅ 等待 2-5 分钟，你的网站就可以通过 `https://你的用户名.github.io` 访问了！

## 2. 如何上传 PDF 文件 📄

### 步骤

1. **把你的 PDF 文件复制到** `f:\aca\assets\pdfs\` 文件夹
   - 例如：你的 CV 命名为 `cv.pdf`，放在这个目录
   - 你的论文命名为 `paper-2026-xxx.pdf`，放在这个目录

2. **在页面中添加链接**

   例如，在 `publications.md` 中：
   ```markdown
   - **论文标题**
     *期刊名称*, 2026
     [[PDF](assets/pdfs/paper-2026-xxx.pdf)]
   ```

3. **提交并推送**
   ```bash
   git add .
   git commit -m "Add paper: 论文标题"
   git push
   ```

### 💡 命名建议

- 使用英文文件名，不要有空格
- 推荐格式：`2026-journal-title.pdf`, `cv-2026.pdf`
- 单个文件不要超过 50MB

## 3. 如何修改网站内容 ✏️

### 修改哪个文件？

| 页面 | 文件 | 说明 |
|------|------|------|
| 首页 | `index.md` | 个人介绍、快速链接 |
| CV | `cv.md` | 简历页面内容 |
| 研究方向 | `research.md` | 研究兴趣、工作论文 |
| 发表论文 | `publications.md` | 论文列表 |
| 教学 | `teaching.md` | 教学经历、课程信息 |
| 联系方式 | `contact.md` | 邮箱、办公室地址 |
| 全局配置 | `_config.yml` | 网站标题、邮箱、社交链接 |

### 示例：修改个人邮箱

打开 `_config.yml`，找到这部分：
```yaml
email: "your.email@example.com"
```
修改为你的真实邮箱。

### 示例：添加一篇新论文

打开 `publications.md`，找到对应的年份，添加：

```markdown
- **我的新论文标题**  
  与 合作者名字  
  *期刊名称*, 2026  
  [[PDF](assets/pdfs/my-new-paper.pdf)]
  [[Google Scholar](链接)]
```

## 4. 日常维护流程 🔄

每次更新网站只需要这 4 步：

### 第 1 步：修改文件
用文本编辑器（Notepad++、VS Code 等）修改对应的 `.md` 文件。

### 第 2 步：确认修改
```bash
git status
```
查看哪些文件被修改了。

### 第 3 步：提交修改
```bash
git add .
git commit -m "更新：添加了 XXX 论文"
```
写清楚这次修改了什么，方便以后查看历史。

### 第 4 步：推送到 GitHub
```bash
git push
```

✅ 完成！刷新网页，几秒钟后就能看到更新了。

## 5. 常见问题解答 ❓

### Q1: 推送时要求输入密码？

A: GitHub 现在不使用密码登录，需要创建 Personal Access Token：
1. GitHub 右上角头像 → Settings → Developer settings → Personal access tokens
2. 生成新 token，勾选 repo 权限
3. 推送时用这个 token 代替密码

### Q2: 网页没有更新？

A: GitHub Pages 构建需要时间：
- 等待 2-5 分钟
- 清除浏览器缓存（Ctrl+F5 强制刷新）
- 去仓库 Settings → Pages 查看状态

### Q3: 如何添加个人照片？

A: 把照片命名为 `profile.jpg`，放到 `assets/images/` 文件夹，然后在 `index.md` 中添加：
```markdown
![我的照片](assets/images/profile.jpg)
```

### Q4: 可以绑定自己的域名吗？

A: 可以！在仓库 Settings → Pages → Custom domain 中设置你的域名，然后在域名服务商那里添加 DNS 记录。

### Q5: PDF 文件太大怎么办？

A: 
- 使用在线 PDF 压缩工具（如 SmallPDF）
- 超过 100MB 的文件 GitHub 会拒绝
- 如果有大量大文件，可以考虑使用云存储链接

### Q6: 如何在本地预览网站？

A: 需要安装 Ruby 和 Jekyll：
1. 安装 Ruby：https://rubyinstaller.org/
2. 安装 Jekyll：`gem install jekyll bundler`
3. 在网站目录运行：`bundle install`
4. 启动本地服务器：`bundle exec jekyll serve`
5. 浏览器访问：http://localhost:4000
